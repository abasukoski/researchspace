Homepage and sidebars

The homepage displays a main template page with two resizable sidebars on both sides.

  • The left vertical sidebar is the clipboard, where resources can be referenced, dragged and visualized in personal folders.
  • The right-hand sidebar is called Details. This is used to show the details of resources. If you click this it will initially be empty. Hovering over a resource card you will see a Details icon. Clicking on it the details sidebar will open and show any data attached to that resource.

Both the details bar and the clipboard bar can be customized to perform different tasks and display different data.

Thinking Frames

On the top bar there is a menu item called “Thinking Frames”. Click this and see a variety of options for working with data.

The base ResearchSpace system works with maps, data graphs, semantic narratives (data embedded into text), entity observation through images, and may other tools. From the clipboard drag a resource card to one of the Thinking Frame option box.

How to use the Clipboard

The clipboard holds resources and folders and opens at the root of those folders. Most resources within ResearchSpace can be dragged to and from a clipboard folder.

The download install has already two entities included: a person and an object. Above these entities there is a folder. This folder has a little diagram symbol to show that it was created from a knowledge map. A normal container folder can be created using the new folder button.

Clipboard Example
  1. Click Thinking Frames and then Entity Editor.
  2. Choose Object. With the clipboard open, add a new folder and use the return key to save your folder name.
  3. Open the new folder.
  4. From the object list (there is only one object) drag it to the folder and release. It is now referenced in your clipboard.

Entity Cards

An entity card is a visualisation of a URI resource. The object ‘Vitruvian man’, for example, has a URI which is of type Human-Made Object. It also has a representation (an image) associated with it – which is automatically rendered in the card. It may also be part of a much wider network of contextual information.

Since the card simply represent a URI it can be passed to different tools and templates to allow further authoring.

The overlay for entities simply provides a link to different tools and template passing the URL as a parameter. For example, clicking on the resource icon the resource template is loaded and immediate relations for that entity displayed.

Entity Resource Template

All the patterns relevant to the entity are displayed in this page through a visualization component. As new relations are added they are display automatically on the template.
